Procedural Maze Generator & Solver

Wait for the maze to finish generating, click on a cell somewhere within the maze, watch the solving algorithm navigate the maze and select the fastest route to the target!

The maze generation occurs in two parts. Firstly, an implementation of the recursive backtracking algorithm runs, generating the raw skeleton. Following this, a function I have added runs to randomly select cells and randomly remove walls which border those cells. The effectiveness of the second phase is controlled by the wall removal factor parameter. If set to zero, no additional walls will be removed, resulting in a maze with exactly one possible path between any two cells. The higher this value, the more open the maze becomes and the greater the number of potential routes between two cells. Play around with the various parameters to explore how both the maze-generation and maze-solver work!

Note: Alterations to the solving settings take effect at the first available static frame. Any other alterations will require generating a new maze.
